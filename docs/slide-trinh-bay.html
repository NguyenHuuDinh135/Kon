Storefront

Danh sách sản phẩm

Trang sản phẩm

Hiển thị toàn bộ catalog sản phẩm với khả năng tìm kiếm và lọc theo danh mục.

  • Tìm kiếm realtime theo tên sản phẩm
  • Lọc theo category (Electronics, Fashion, Home...)
  • Responsive grid: 1→2→3→4 cột theo viewport
  • Lazy loading ảnh (performance optimization)
  • Gọi API: GET /products?search=&limit=50
Xác thực

Đăng nhập & Phân quyền

Hệ thống xác thực JWT

Bảo mật bằng JSON Web Token với bcrypt password hashing.

  • Form đăng nhập: username + password
  • API: POST /auth/login → trả về JWT token
  • Token lưu localStorage + cookie (SSR support)
  • 2 role: Admin (full dashboard) và Client (storefront)
  • Rate limiting: 5 lần/phút chống brute-force
  • Middleware redirect: chưa login → /login

Machine Learning

Dự báo doanh thu (Linear Regression)

Revenue Forecast Chart

Dự phóng doanh thu 3 tháng tiếp theo bằng hồi quy tuyến tính trên dữ liệu Olist.

  • Đường solid (xanh): Doanh thu thực tế (historical)
  • Đường dashed (cam): Dự báo 3 tháng tới
  • Model: LinearRegression trên monthly aggregated revenue
  • Metric: R² score hiển thị trên chart
  • API: GET /analytics/forecast
  • Auto-retrain mỗi 4 giờ → forecast luôn cập nhật
Machine Learning

Phân nhóm khách hàng (K-Means)

1

Scatter Plot RFM

Biểu đồ phân tán 3D: Recency × Frequency × Monetary. Mỗi màu = 1 cluster. 4,300+ khách hàng được phân thành 5 nhóm.

2

5 Segments

VIP (high F+M, low R), Loyal (medium-high), Regular (trung bình), At Risk (tăng R), Lost (rất cao R, thấp F).

3

Cluster Statistics

Bảng thống kê mỗi cluster: số lượng, RFM trung bình, silhouette score. API: GET /predictions/clustering.

Machine Learning

Dự đoán Engagement (Decision Tree)

Decision Tree + GridSearchCV

Phân loại mức độ tương tác khách hàng thành 4 level.

  • Output: Low / Medium / High / VIP engagement
  • Features: Tenure, Satisfaction, DeviceRegistered, WarehouseToHome...
  • Tuning: GridSearchCV trên max_depth, min_samples
  • Feature Importance: Bar chart hiển thị feature nào ảnh hưởng nhất
  • Confusion Matrix: Ma trận nhầm lẫn đánh giá accuracy
  • API: GET /predictions/decision-tree
Machine Learning

Dự đoán Churn (Logistic Regression)

1

Churn Probability

Mỗi khách hàng có xác suất rời bỏ 0-100%. Phân 3 mức: HIGH (>70%), MEDIUM (30-70%), LOW (<30%). Dữ liệu huấn luyện: 5,630 khách có label thực.

2

ROC Curve

Đường cong ROC đánh giá khả năng phân biệt churn/không churn. AUC càng gần 1 càng tốt. Hiển thị realtime sau mỗi lần retrain.

3

SHAP Values

Giải thích AI: feature nào đóng góp nhiều nhất vào dự đoán churn? Tenure, DaySinceLastOrder, Complain thường top 3.

4

Risk Distribution

Biểu đồ phân bố số lượng khách theo mức rủi ro. Cảnh báo tự động khi nhóm HIGH tăng đột biến.

Machine Learning

So sánh các mô hình ML

Model Comparison Dashboard

So sánh kết quả dự đoán của tất cả ML models cho từng khách hàng.

  • K-Means: Khách thuộc cluster nào (VIP→Lost)
  • Decision Tree: Engagement level + confidence %
  • Logistic Regression: Churn probability %
  • Bảng so sánh side-by-side cho mỗi CustomerID
  • Metrics tổng hợp: Accuracy, F1, ROC-AUC của từng model
  • API: GET /predictions/compare
AI Agent

Trợ lý AI Agent (LangGraph)

1

Chat Interface

Giao diện chat realtime với AI. Hỗ trợ Markdown, bảng, code. Streaming response (hiển thị từng token).

2

7 Agent Tools

Agent tự chọn tool phù hợp: query_database, get_customer_profile, get_churn_risk_summary, get_revenue_insights, suggest_campaign, search_similar_customers, get_product_recommendations.

3

ReAct Pattern

Reasoning → Action → Observation loop. Agent suy nghĩ, chọn tool, đọc kết quả, suy nghĩ tiếp → trả lời bằng tiếng Việt.

4

LLM Backend

Primary: AWS Bedrock Claude Haiku (fast, accurate). Fallback: Ollama qwen2.5 (local, offline). Tự động chuyển đổi.

AI Agent

Tìm kiếm ngữ nghĩa (pgvector)

Vector Similarity Search

Tìm khách hàng bằng mô tả tự nhiên thay vì filter cứng.

  • Input: "khách chi tiêu nhiều nhưng lâu không quay lại"
  • Process: Text → Embedding 1024-dim → Cosine similarity
  • Embedding model: Titan Embed v2 (Bedrock) hoặc mxbai-embed-large (Ollama)
  • Database: pgvector extension, HNSW index
  • Output: Top K khách hàng tương tự nhất
  • API: GET /search/behavior?q=...
Tính năng

Quản lý chiến dịch Marketing

AI-Generated Campaigns

Chiến dịch marketing được AI đề xuất tự động dựa trên phân tích churn.

  • Tự động tạo: Autonomous loop phát hiện nhóm rủi ro cao → tạo draft campaign
  • Human-in-the-loop: Admin duyệt trước khi thực thi
  • Workflow: Draft → Approved → Executed
  • Targeting: Theo segment (high_churn, vip, at_risk)
  • Channels: Email, Push notification, SMS
  • APIs: GET/POST /campaigns, PUT /campaigns/:id/approve
Quản lý

Khách hàng & Đơn hàng

Quản lý khách hàng

Bảng 5,630 khách hàng với thông tin churn risk realtime.

  • Hiển thị: ID, Gender, Tenure, Satisfaction, Orders, Churn%
  • Filter theo risk level (High/Medium/Low)
  • Badge màu theo mức rủi ro (đỏ/vàng/xanh)
  • Click vào row → xem chi tiết profile

Quản lý đơn hàng

100K+ đơn hàng từ Olist với trạng thái và tổng tiền.

  • Columns: Order ID, Customer, Total, Status, Date
  • Status badges: delivered, shipped, processing, cancelled
  • Pagination (50 records/page)
  • API: GET /orders?limit=50
Tính năng

Thông báo tự động & Sản phẩm

Hệ thống thông báo

Notifications được tạo tự động bởi Autonomous Loop khi phát hiện anomaly.

  • Trigger tự động: Khách churn>70%, revenue drop>15%
  • Unread badge: Số thông báo chưa đọc trên sidebar
  • Mark all read: Đánh dấu tất cả đã đọc
  • API: GET /notifications, PUT /notifications/read-all
Backend

ETL Pipeline & Auto ML Training

Worker Service (Background)

Service chạy nền xử lý toàn bộ data pipeline và ML training.

  • ETL: 3 datasets Kaggle → Clean → Load PostgreSQL
  • ML Train: 5 models train tuần tự (K-Means → DT → LR → LinReg → CF)
  • Schedule: APScheduler retrain mỗi 4 giờ
  • Embeddings: Generate 1024-dim vectors cho 200 customers
  • Autonomous Loop: OAPA cycle sau mỗi lần train
  • Memory limit: 2GB (ML training intensive)
DevOps

CI/CD Pipeline (GitHub Actions)

Automated Testing & Deployment

Mỗi push lên master tự động chạy pipeline kiểm tra chất lượng code.

  • Step 1: ESLint — kiểm tra coding style frontend
  • Step 2: TypeScript check — đảm bảo type safety
  • Step 3: pytest — chạy API integration tests
  • Step 4: Playwright — E2E browser tests
  • Trigger: Push to master, Pull Requests
  • Runner: ubuntu-latest (GitHub hosted)
